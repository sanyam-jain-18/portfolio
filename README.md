# sanyamjain.xyz — personal portfolio

Personal site and writing surface for Sanyam Jain — senior full-stack engineer
(frontend center of gravity). Case studies for production work, a few
self-directed projects, and a downloadable ATS-friendly resume generated on
the fly from source data.

Live at **[sanyamjain.xyz](https://sanyamjain.xyz)**.

## Stack

- **Next.js 16** (App Router, Turbopack) + **React 19** + **TypeScript**
- **Tailwind v4** (CSS-first, `@theme` tokens)
- **MDX** via `@next/mdx` — case studies live in `src/content/work/*.mdx`
  and import inline React diagram components
- **Recharts v3** + bespoke SVG components for case-study visuals
- **@react-pdf/renderer** — resume PDF generated server-side from a single
  source-of-truth (`src/lib/resume.ts`), served via a route handler
- **motion** (framer-motion v12) for the reveal/stagger primitives
- **next-themes** for theme switching
- Deployed on **Vercel** (Hobby tier)

## Local development

Requires **Node 22+** (enforced via `engines` in `package.json` and
respected by Vercel).

```bash
npm install
npm run dev          # http://localhost:3000
```

The resume PDF is generated by a route handler at `/api/resume` — visit it
locally to preview without downloading. The download button on the About
page wires straight to that endpoint.

```bash
npm run lint         # eslint
npm run build        # production build
npm run start        # serve the production build locally
```

## Project layout

```
src/
  app/                       # App Router pages + API routes
    api/resume/route.tsx     # PDF route handler — renders ResumePdfDocument
    about/page.tsx           # About + resume preview + contact
    work/[slug]/page.tsx     # MDX-driven case study pages
  components/
    diagrams/                # Per-case-study SVG / Recharts diagrams
    resume-pdf-document.tsx  # @react-pdf/renderer document
    resume-section.tsx       # HTML resume preview (mirrors the PDF)
  content/work/              # *.mdx case studies — imports diagram components
  lib/
    site.ts                  # name, role, socials, copy
    resume.ts                # single source of truth for the resume
    work.ts                  # case-study registry + card visuals
    projects.ts              # "Other Work" registry
```

A change to `src/lib/resume.ts` updates **both** the on-page preview and the
generated PDF — they read the same data.

## Deploying changes

Vercel auto-deploys on push to `main`. The flow:

```bash
# 1. make the change locally, verify in `npm run dev`
git add <files>
git commit -m "feat(scope): describe the change"
git push origin main
```

Vercel picks up the push, runs `next build`, and rolls the new build to
production within ~2 min. Preview deploys are created automatically for any
non-`main` branch pushed to the remote.

If a deploy fails:

1. Check the build log in the Vercel dashboard → Deployments
2. Most failures are TypeScript errors that didn't surface locally — run
   `npm run build` locally to reproduce
3. The previous successful deploy stays live until the new one succeeds, so
   production is never broken by a bad push
